About Princess

A lightweight, extendable CSS library built with Sass and Normalize that's blissfully easy to customize.

Princess is inspired by Pure's simplicity and Blueprint's flexibility.

The idea behind Princess is that you receive a set of base styles of primary elements, along with modules that include classes for optional styling, all based on Normalize and two files containing variables. Princess is written in Sass for easy customization and can be used in whole or in parts.

A lot of me has gone into Princess. I like whitespace, big fonts, and smooth CSS transitions. There's even a little module for improving code snippet highlighting in Jekyll! Princess was written because I wanted something I could use for all of my future projects that was already customized and styled to fit my needs. Because I love writing documentation, I decided to package it up for distribution.

The documentation for Princess is styled using Princess.

What it is not

  • A responsive grid
  • Opinionated
  • Bloated
  • JavaScript

What it is

  • Cross-browser friendly
  • Easy to customize
  • Easy to extend
  • Semantically named and organized
  • Modular

Documentation

The guides are organized according to which section of the Princess library each element belongs to.

  • Styles that are automatically applied to your content are included in the Core group.
  • Variables includes the list of variables used in Princess.
  • To learn about customizing Princess or only using specific modules, browse the Customization docs.
  • Modules are isolated to special classes that you have to call to use. Things like buttons, tables, and form elements live here.
  • Menus, headers, footers, and other layouts are covered in Components.

Read the first item in each subsection. It contains vital information about that items in that group, and the rest of the documentation assumes you've read it. Generally you're presented with a list first (Variable list, Module list, etc) or a guide to that section (usage, basics).
